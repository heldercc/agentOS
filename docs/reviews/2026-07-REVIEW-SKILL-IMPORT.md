# Review — Skill Import from claude-skills (alirezarezvani, MIT, 355 skills)

Session 004. Mandate from the Pilot: validate what is good and import it,
discard the rest — devil's advocate mode. Verdicts per the five-verdict
taxonomy. Acceptance rate: 2 of 355 (0.6%) — by design, not stinginess:
the engine's value is the Pilot's differentiated judgment; commodity
expertise is imported only when a concrete near-term use exists (Article 8).

## Accepted → expertise/imported/
- **reflect** — a governed "step back and reassess direction" mechanism.
  Native fit for Executive Mode and for our own working sessions; pure
  reasoning, no dependencies, high craft. First citizen of origin `imported`.
- **capture** — brain-dump → structured, zero-loss triage. The Pilot
  generates ideas faster than any process absorbs them (empirically proven
  today); this is the natural intake ahead of the Incubator and a future
  Composer mode.

## Rejected
- **handoff** — session continuity done poorer than Navigation; importing a
  weaker duplicate of our own doctrine would be self-vandalism. Recorded as
  independent validation that the problem is real.
- **process-mapper** — Lean/ToC/cycle-time canon of genuine quality, but the
  Pilot is an industrial engineer: importing what the owner already masters
  produces shelf-ware. Revisit only if a vertical needs it packaged.
- **orchestration/, loop-library/** — someone else's architecture; ours has
  planes and a Constitution. Read for inspiration in Book II design, never
  imported.
- **c-level personas (CFO/CMO/CTO...)** — doctrinal violation: personas
  conflate identity with competence (ADR-0002). Hard reject.
- **engineering-team agents** — redundant with the coding Copilot itself.
- **marketing / finance / compliance packs** — Phase 5 material; hoarding
  now is complexity without justification. The repo will still exist when a
  vertical needs it; import then, through the gate.

## Incubator (insights harvested, skills discarded)
- **Seed hygiene** (from knowledge-ops) — its SOP failure modes are exactly
  seed-store failure modes: no owner, no last-reviewed date, vague success
  signals, orphaned entries, glossary drift. The seed lifecycle ADR (owed
  before Phase 6) inherits this checklist.
- **Skill Import as Composer mode** — the SKILL.md standard (agentskills.io)
  is ~80% compatible with GuruSeed anatomy; a Composer "import" flow is
  near-free. Design in Phase 2.
